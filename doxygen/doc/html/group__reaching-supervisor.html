reaching-supervisor

Module to handle the communcation between reaching-planner and reactController. Version:1.0

Description

This module contains rpc service for user interface (see service part)

To correctly use the module with iCub Simulator, do the following:

  • Launch iCub Simulator, i.e typing iCub_SIM in a terminal
  • Launch reactController module form a new terminal
  • Launch reaching-planner module
  • Launch this supervisor module
  • Connect to the service port of this module, i.e. typing yarp rpc "/module_name/rpc:i"

To correctly use the module with real robot, do the following:

  • Launch WYSIWYD application from yarpmanager
  • Launch actionsRenderingEngine module from a new terminal
  • Connect all port in WYSIWYD application from yarpmanager
  • Open another new terminal, put some objects on the table in front of the iCub face and type:
    • echo "home all" | yarp rpc /actionsRenderingEngine/cmd:io, to move iCub to home pose.
    • echo "calib table" | yarp rpc /actionsRenderingEngine/cmd:io, to calibrate the table with iCub's finger.
  • Launch reactController module form a new terminal
  • Launch reaching-planner module
  • Launch this supervisor module
  • Connect to the service port of this module, i.e. typing yarp rpc "/module_name/rpc:i"

Parameters

  • –rate: Period of the multiple particles generation (in ms).
  • –verbosity: Verbosity.
  • –tol: Tolerance of the multiple particles generation (in m).
  • –speedEE: Speed of End-Effector motion (in m/s).
  • –localPlanningTime: Planning time of each local planner (in s).
  • –globalPlanningTime: Planning time of the whole planner (in s).
  • –targetName: Name of target of the planner.

Input Ports

  • /module_name/bestCartersianTrajectory:i [waypointTrajectory] [default carrier:tcp]: receive trajectories of controlled points in the format: (((control-point string_value) (number-waypoints int_value) (number-dimension int_value) (waypoint_0 (x0 y0 z0)) waypoint_1 (x1 y1 z1)) ...) ( controlled point 2 ) ...)
  • /module_name/rpc:i [yarp::os::Bottle] [default carrier:tcp]: receive user command.

Output Ports

  • /module_name/reaching-planner/rpc:o [yarp::os::Bottle] [default carrier:tcp]: send request of replanning to reaching-planner.
  • /module_name/particlesCartersianTrajectory:o [waypointTrajectory] [default carrier:tcp]: send particled trajectories of controlled points in the format: (((control-point string_value) (number-waypoints 1) (number-dimension int_value) (waypoint_0 (x0 y0 z0)) waypoint_1 (x1 y1 z1)) ...) ( controlled point 2 ) ...)

Services

  • /module_name/rpc:i [rpc-server]: service port . This service is described in reachingSupervisor_IDL (idl.thrift)
